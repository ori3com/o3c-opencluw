## 2025-04-26 - Optimized Chat Markdown Parsing
**Learning:** In highly trafficked text parsing methods, such as `stripInboundContextBlocks` in `ChatMarkdownPreprocessor.kt`, using `split("\n")` followed by collecting to an intermediate `mutableListOf<String>` and calling `.joinToString("\n")` creates a massive amount of intermediate String and Collection allocations per message.
**Action:** Use `.lineSequence()` for lazy evaluation of multi-line strings, and stream valid lines directly into a pre-allocated `StringBuilder` manually. Additionally, replace simple trailing/leading text modifications using `Regex.replace()` with standard library string functions like `.trimStart('\n')` to entirely skip Regex compilation and state machine overhead.
