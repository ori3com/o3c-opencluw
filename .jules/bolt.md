## 2024-04-24 - Optimize String Splitting in ChatMarkdownPreprocessor
**Learning:** The `stripInboundContextBlocks` method in `ChatMarkdownPreprocessor.kt` was allocating multiple intermediate strings and lists by using `replace("\r\n", "\n")`, `split("\n")`, and `joinToString("\n")`. This created substantial memory allocation and garbage collection pressure for large chat markdown payloads.
**Action:** Replaced string splitting allocations with `lineSequence()` to lazily iterate over lines and directly appended them to a pre-sized `StringBuilder`. This prevents allocating an intermediate list of strings, removing string allocation overhead from `split` and `joinToString`.
