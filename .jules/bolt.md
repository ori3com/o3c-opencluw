## 2023-10-27 - TTS Split Text GC Thrashing
**Learning:** High-frequency string processing tasks like TTS chunking can cause severe GC thrashing due to continuous $O(N^2)$ `substring` allocations. Additionally, when searching backwards for delimiters within chunks, `lastIndexOf` naturally sweeps backwards through the entire string to index 0, which scales poorly when delimiters are missing. The codebase was also doing this redundantly across two classes.
**Action:** When working on string parsing or splitting features, always prefer tracking `offset` and `limit` bounds instead of allocating intermediate string chunks. When doing reverse-bounded string searches, use a custom `regionMatches` loop to tightly constrain the search window instead of standard `lastIndexOf`. Ensure the solution is centralized in utility classes to prevent duplicate unoptimized implementations.
