## 2024-05-19 - TTS Text Splitting Array Optimization
**Learning:** In `TTSUtils.kt`, replacing `listOf` with `arrayOf` for top-level array-like constants (`SENTENCE_ENDERS`, `COMMA_ENDERS`) helps avoid iterator allocations during tight loop chunks. However, refactoring the `findBestSplitPoint` to use bounding variables (`offset`, `searchLimit`) for the full `text` backwards search using `text.lastIndexOf` causes a severe O(N^2) regression in Kotlin because `lastIndexOf` searches all the way back to index 0, rather than strictly stopping at the `offset`. The original string substring method, while allocating strings, inherently bounds the backward search. Reverted the offset search bounding to keep bounded search time.
**Action:** Always be wary of replacing `substring()` bounding with offset indices if using `lastIndexOf()` in Kotlin/Java without an explicit `startIndex` that acts as a true hard-limit rather than just where the backwards search *begins*.
