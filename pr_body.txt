💡 **What**: Replaced a `joinToString("\n") { it.text ?: "" }` call inside `toUiChatMessage()` in `ChatViewModel.kt` with a manual `StringBuilder` and a `for` loop over indices.

🎯 **Why**: In Android (and Kotlin generally), `joinToString` on a generic collection creates an intermediate string iterator, implicit list mapping elements, and causes multiple string builder lambda allocations. In a highly-invoked UI mapping function (like `toUiChatMessage` which runs constantly as new chat messages are pushed or appended during list reconciliation), this creates immense, unnecessary pressure on the Garbage Collector (GC), leading to CPU overhead and potential UI stuttering (jank).

📊 **Impact**: Prevents implicit lambda allocation and intermediate string instantiation during every message update mapping cycle, directly reducing GC pauses and jitter in the chat scroll stream.

🔬 **Measurement**: Verified using local unit tests (`app:testStandardDebugUnitTest`) and lint checks to ensure strict functionality parity (including correct edge-case handling for blank/null strings falling back to "(thinking)").
